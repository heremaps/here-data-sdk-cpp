Parameters
[in]requestRequest to be sent.
[in]payloadStream to store response payload data.
[in]callbackCallback to be called when request is fully processed or cancelled. After this call there will be no more callbacks triggered and users can consider the request as done.
[in]header_callbackCallback to be called when a HTTP header has been received. Each HTTP header entry will result in a callback.
[in]data_callbackCallback to be called when a chunk of data is received. This callback can be triggered multiple times all prior to the final Callback call.
Returns
SendOutcome which represent either a valid RequestId as the unique request identifier or a ErrorCode in case of failure. In case of failure no callbacks will be triggered.
